Трассировка HTTP сеанса.

 

Протокол HTTP определяет, каким образом клиенты (например, браузеры) запрашивают web-страницы, а серверы осуществляют передачу этих страниц. Когда пользователь запрашивает web-страницу (например, совершает щелчок на гиперссылке), браузер посылает серверу HTTP-запрос объектов, составляющих web-страницу. Сервер получает запрос и высылает ответные сообщения, содержащие требуемые объекты.

Для трассировки соединения HTTP следует настроить соответствующим образом утилиту Network Monitor и файловый менеджер Total Commander, как это описано в пункте "Настройка утилиты Network monitor". Далее следует запустить сбор пакетов в Network Monitor'е. Для того, чтобы проследить сам процесс сеанса HTTP передачи данных, необходимо с помощью браузера открыть файл info[1].htm на сервере TITAN.VT, а затем остановить сбор пакетов. Далее можно настроить фильтр NetMon'а для того, чтобы он не отрображал пакеты, не относящиеся к протоколам TCP и HTTP.

 

Фильтрация HTTP сеанса.

Далее необходимо установить фильтр для того, чтобы не отследживать обмен с другими узлами. Для этого необходимо выбрать пункт "Filter" меню "Capture".Для того, чтобы Network Monitor мог отследить передачу по протоколу HTTP следует добавить еще один фильтр. Для этого нужно выделить недавно добавленное выражение и нажать на кнопку "OR", а затем снова "Expression". В открывшемся окне необходимо перейти на вкладку "Protocol", нажать кнопку "Disable All", после чего все протоколы будут перемещены в правую панель, и разрешить протокол "HTTP".
