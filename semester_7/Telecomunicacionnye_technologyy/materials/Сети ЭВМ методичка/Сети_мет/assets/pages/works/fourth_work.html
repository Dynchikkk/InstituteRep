Протоколы прикладного уровня

Протокол HTTP

Изучение и анализ протоколов прикладного уровня с помощью диагностических утилит операционной системы Windows и программных средств, позволяющих контролировать сетевой трафик на примере программы Network Monitor.

HTTP (HyperText Transfer Protocol — протокол передачи гипертекста) - протокол прикладного уровня в модели TCP/IP. Протокол предназначен для передачи файлов и отображения их на стороне клиента. Протокол представляет из себя связь клиента и сервера: клиент посылает запрос - сервер отвечает. Между клиентом и сервером находится прокси-сервер (proxy), он позволяет сохранить анонимность клиента, так как выполняет пользовательский запрос от своего имени, выступает, своего рода, посредником.

    Механизм работы протокола следующий:
  1. Пользователь указывает необходимый ему URL (Uniform Resource Locator) - уникальный адрес интернет-ресурса.
  2. Браузер сопоставляет URL с IP-адресом сервера с помощью DNS (Domain Name System, система доменных имён) - системы связывания доменных имён и IP-адресов и посылает HTTP-запрос о подключении.
  3. Сервер посылает ответ с указанием версии протокола, статуса подключения и дополнительных сведений.

HTTP-запросы клиента

Запросы осуществляют связь между клиентом и сервером и могут посылаться только клиентом.

    Существует три вида запросов, они начинаются с разных ключевых слов:
  1. GET - отправленные сообщения содержат только URL и, в некоторых случаях, несколько параметров. Сервер обрабатывает запрос и отображает результат на веб-странице
  2. POST - необязательные параметры помещаются в тело сообщения запроса.
  3. HEAD - отправленные сообщения содержат только URL и, в некоторых случаях, несколько параметров. Сервер обрабатывает запрос и отображает результат внутри тега в разметке веб страницы.
    Запрос выглядит следующим образом:
  1. GET/index.html HTTP/1.1
  2. Host: www.dasha.com
    А ответ так:
  1. HTTP/1.1 200 ОК
    Цифры в ответе являются статусом запроса, например:
  1. 200 OK - загрузка страницы произошла успешно;
  2. 300 Moved Permanently - URL-адрес ресурса изменён навсегда;
  3. 400 Bad Request - запрос сформирован неверно;
  4. 404 Not Found - запрашиваемый ресурс не найден.

Задание

Создать и настроить Capture в программе Network Monitor. Как это сделать, написано в теоретическом материале. Так же необходимо учитывать тот факт, что в Network Monitor ответ назван запросом.

Для корректного выполнения работы, обратите внимание на запросы, отображаемые браузером. Для этого:

  1. Нажать на клавиатуре F12 или сочетание клавиш Fn+F12;
  2. В главном меню выбрать Network и выбрать запрос;
  3. В открывшемся окне будет представлена полная информация о кадре.

Изучить работу HTTP-протокола

    Для этого:
  1. Запустить окно захвата в сетевом мониторе;
  2. Открыть браузер и зайти на любой сайт, совершить несколько переходов, закрыть вкладку с сайтом и закрыть браузер;
  3. Остановить захват Capture;
  4. Отфильтровать запросы и оставить только HTTP-кадры, представить разбор нескольких кадров в отчёте. Для фильтрации необходимо ввести «http or tcp» в поле «Display Filter» и нажать на кнопку «Applay».

Анализ заголовка

Проанализировать заголовки запросов и ответов и представить кадры с различными флагами в отчёте.

Анализ тегов запроса

Просмотреть код страницы и представить в отчёте теги, которые вызвали запрос к веб-серверу и ответ сервера. Для просмотра кода страницы необходимо нажать сочетание клавиш CTRL+U или нажать правой кнопкой мыши по странице и выбрать «Посмотреть код страницы».
