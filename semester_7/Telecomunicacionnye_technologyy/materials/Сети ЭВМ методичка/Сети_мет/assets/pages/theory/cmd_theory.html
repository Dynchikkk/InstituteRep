Работа с командной строкой Windows

Командная строка в ОС Windows (cmd, терминал, консоль) — это ПО, которое обеспечивает взаимосвязь между пользователем и операционной системой с помощью текстовых команд.

Как использовать командную строку

В терминале необходимо вводить команды, в противном случае вы увидите уведомление о том, что команда нераспознана.

Команды имеют следующий синтаксис:

команда -[параметр [аргумент]]. Параметров может быть несколько, в этом случае они вводятся через пробел, перед параметром ставится дефис (-) или слэш (/). Параметры могут быть обязательными и необязательными: необязательные указываются по желанию, без обязательных параметров вы получите справочную информацию о команде.

Для прекращения работы команды необходимо нажать на клавиатуре сочетание клавиш Ctrl+C.

Для того, чтобы повторно использовать ранее введённую команду, необходимо нажать на клавиатуре стрелку вверх. Для перемещения в тексте команды необходимо так же использовать стрелки.

Для получения информации о командах, которые вы можете использовать, необходимо набрать help, затем нажать Enter.

Для получения справочной информации о конкретной команде, необходимо ввести её в консоль и добавить [/?]

Ping

Позволяет проверять собственную связь и состояние удалённых хостов, находить и локализировать неполадки в программном и аппаратном обеспечении, тестировать качество сетей.

Удалённый хост можно вводить в виде ip адреса или использовать его доменное имя. Хост является обязательным.

    ping [-t] [-a] [-n <число>] [-l <размер>] [-f] [-i ]
  • [-v ] [-r <число>] [-s <число>]
  • [[-j <список_узлов>] | [-k <список_узлов>]]
  • [-w <время_ожидания>] [-R] [-S <адрес_источника>]
  • [-c секция] [-p] [-4] [-6] конечный_узел
    Наиболее часто используемые параметры:
  1. /? — Вывод справочного сообщения
  2. -t — Проверяет связь с указанным узлом до прекращения.
  3. Для отображения статистики и продолжения проверки нажмите клавиши CTRL+BREAK;для прекращения нажмите CTRL+C.
  4. -a — Позволяет определить доменное имя компьютера по его адресу.
  5. -n <число> — Число отправляемых запросов проверки связи (пакетов Echo). Количество по умолчанию - 4 пакета
  6. -l <размер> — Размер буфера отправки (сообщения).
  7. -f — Устанавливает флаг, запрещающий фрагментацию, в пакете (только IPv4).
  8. -i — Срок жизни пакетов.
  9. -w <время_ожидания> — Задает время ожидания каждого ответа (в миллисекундах).
  10. -4 — Задает принудительное использование протокола IPv4.
  11. -6 — Задает принудительное использование протокола IPv6.

Tracert

Tracert позволяет выяснить последовательность маршрутизаторов IP пакета на пути к конечному узлу. отправляет на хост назначения ICPM-запрос «Echo Request» с TTL=1.

Удалённый хост можно вводить в виде ip адреса или использовать его доменное имя. Хост является обязательным.

tracert [-d] [-h максЧисло] [-j списокУзлов] [-w таймаут] [-R] [-S адресИсточника] [-4] [-6] конечноеИмя
    Наиболее часто используемые параметры:
  1. /? — Вывод справочного сообщения
  2. -d — Без разрешения в имена узлов.
  3. -h максЧисло — Максимальное число прыжков при поиске узла (по умолчанию - 30).
  4. -j списокУзлов — Свободный выбор маршрута по списку узлов (только IPv4).
  5. -w таймаут — Таймаут каждого ответа в миллисекундах.
  6. -R — Трассировка пути (только IPv6).
  7. -S адресИсточника — Используемый адрес источника (только IPv6).
  8. -4 — Принудительное использование IPv4.
  9. -6 — Принудительное использование IPv6.

Arp

Arp позволяет просмотреть или изменить записи в кэше ARP (Address Resolution Protocol - протокол разрешения адресов).

  1. ARP -s inet_addr eth_addr [if_addr]
  2. ARP -d inet_addr [if_addr]
  3. ARP -a [inet_addr] [-N if_addr] [-v]
  1. eth_addr — Определяет физический адрес.
  2. if_addr — Если параметр задан, он определяет адрес интерфейса в Интернете, чья таблица преобразования адресов должнаизмениться. Если параметр не задан, будет использован первый доступный интерфейс.
  3. inet_addr — Определяет IP-адрес.
    Наиболее часто используемые параметры:
  1. /? — Вывод справочного сообщения
  2. -a — Отображает текущие ARP-записи, опрашивая текущие данные протокола. Если задан inet_addr, то будут отображены IP и физический адреса только для заданного компьютера. Если ARP используют более одного сетевого интерфейса, то будут отображаться записи для каждой таблицы.
  3. -g — То же, что и параметр -a.
  4. -v — Отображает текущие ARP-записи в режиме подробного протоколирования. Все недопустимые записи и записи в интерфейсе обратной связи будут отображаться.
  5. -N if_addr — Отображает ARP-записи для заданного в if_addr сетевого интерфейса
  6. -d — Удаляет узел, задаваемый inet_addr. Параметр inet_addr может содержать знак шаблона * для удаления всех узлов.
  7. -s — Добавляет узел и связывает адрес в Интернете inet_addr с физическим адресом eth_addr. Физический адрес задается. Пример использования утилиты для просмотра arp кэша:

Netstat

Предназначена для получения сведений о состоянии сетевых соединений и слушаемых на данном компьютере портах TCP и UDP, а также, для отображения статистических данных по сетевым интерфейсам и протоколам.

netstat [-a] [-b] [-e] [-f] [-n] [-o] [-p протокол] [-r] [-s] [-t] [-x] [-y] [интервал]

    Наиболее часто используемые параметры:
  1. /? — Вывод справочного сообщения
  2. -a — Отображает текущие ARP-записи, опрашивая текущие данные протокола. Если задан inet_addr, то будут отображены IP и физический адреса только для заданного компьютера. Если ARP используют более одного сетевого интерфейса, то будут отображаться записи для каждой таблицы.
  3. -b — Отображение исполняемого файла, участвующего в создании каждого подключения или порта прослушивания.
  4. -e — Отображение статистики Ethernet. Может применяться вместе с параметром -s.
  5. -n — Отображение адресов и номеров портов в числовом формате.
  6. -o — Отображение ИД процесса каждого подключения.
  7. -r — Отображение таблицы маршрутов.
  8. interval — Повторное отображение выбранной статистики с паузой между отображениями, заданной интервалом в секундах. Чтобы прекратить повторное отображение статистики, нажмите клавиши CTRL+C. Если этот параметр опущен, netstat напечатает текущую информацию о конфигурации один раз.

Nslookup

Инструмент сетевого администрирования для запросов в доменной системе имен (DNS) с целью получения доменного имени, IP-адреса или другой информации из записей DNS. Кроме того, эта команда используется для поиска и устранения проблем с DNS.

  1. nslookup [-opt ...] — интерактивный режим с использованием сервера по умолчанию
  2. nslookup [-opt ...] — server — интерактивный режим с использованием сервера "server"
  3. nslookup [-opt ...] — поиск узла "host" с использованием сервера по умолчанию
  4. nslookup [-opt ...] — поиск узла "host" с использованием сервера "server"
