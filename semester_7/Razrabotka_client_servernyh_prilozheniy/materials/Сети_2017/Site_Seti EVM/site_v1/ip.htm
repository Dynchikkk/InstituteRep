Сетевой протокол

        Сетевой уровень протокола TCP/IP выполняет диспетчерские функции для пакетов данных Internet и базируется на протоколе IP.

        Адрес сети Internet (IP-адрес) имеет длину 32 бита и обычно записывается в формате dotted-decimal (десятичное с точкой), например 192.102.249.3. IP-адрес скрывает за собой номера сети и сетевого интерфейса. Распределением IP-адресов компьютерным сетям занимается Информационный центр сети Internet (Internet Network Information Center - InterNIC). В зависимости от значения старших бит все IP-адреса разделяются на классы, приведенные в таблице 1.

        Номера сетей присваиваются администрацией InterNIC, а номера компьютеров в сетях - непосредственно сетевыми администраторами. В целях увеличения эффективности одна сеть может разделяться на несколько подсетей путем деления адресного пространства.

          Протокол IP - это ненадежный, не ориентированный на соединение протокол, пользующийся датаграммами для доставки информации в сети TCP/IP. IP-датаграмма представляет собой самодостаточный блок данных, состоящий из заголовка и собственно данных. Формат IP-датаграммы представлен на рис. 4.
 


Рис. 4. Формат IP-датаграммы


 


        Описание полей заголовка IP-датаграммы приводится ниже:

  • номер версии - версия IP-протокола, создавшего данную датаграмму;
  • длина заголовка - длина заголовка в 32-битных словах;
  • тип службы - набор приоритетов датаграммы, используемый для организации ее доставки;
  • длина пакета - длина датаграммы в байтах, включая заголовок;
  • идентификатор фрагмента - уникальный идентификатор, присвоенный датаграмме для связывания фрагментированных пакетов;
  • флаги - тип текущего фрагмента датаграммы;
  • смещение фрагмента - смещение текущего фрагмента от начала датаграммы в 64-битных словах;
  • время существования - срок существования датаграммы в сети: при прохождении датаграммы через любой маршрутизатор значение данного поля уменьшается на величину задержки датаграммы внутри буфера в секундах, при нулевом значении данного поля датаграмма уничтожается и посылается соответствующее извещение источнику;
  • протокол - тип протокола-источника данных, инкапсулированных в датаграмму:

    •             1 - ICMP;
                2 - IGMP;
                6 - TCP;
                17 - UDP;
  • контрольная сумма заголовка - результат дополнения до единицы суммы 16-разрядных слов заголовка;
  • IP-адрес источника и получателя - IP-адрес сетевого интерфейса отправителя и получателя данных;
  • опции - поле, используемое для тестирования и отладки сетевых приложений, а также для управления фрагментацией и маршрутизацией сетевых пакетов;
  • заполнение - поле, необходимое для выравнивания поля данных.
