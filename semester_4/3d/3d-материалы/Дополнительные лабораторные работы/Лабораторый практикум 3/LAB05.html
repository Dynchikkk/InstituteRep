Теоретическая часть

Для решения задачи заполнения полигонов разработано множество алгоритмов, в том числе алгоритмы с "затравочным" пикселом, когда вблизи границ полигона генерируются случайным образом координаты точки. В том случае, если специальным тестированием устанавливается принадлежность точки внутренней области полигона, она считается "затравкой", а соседние с ней пикселы последовательно окрашиваются требуемым цветом вплоть до граничных пикселов полигона.

Алгоритм на основе заполнения "по ребрам" не относится к алгоритмам с "затравкой", однако для него характерно сочетание высокой эффективности и простоты. Его суть состоит в том, что для каждой сканируемой строки, пересекающей ребро полигона в точке, (Xi, Yi)   заполняются требуемым цветом все пикселы, центры которых находятся справа от (Xi, Yi), т.е. у которых (X + 0.5) > Xi. На рис .1.а, представлена реализация данного требования для первого выбранного ребра полигона Р2-Р3.

Во втором шаге рис .1.б, выбирается следущее по порядку обхода ребро Р3-Р4. При этом цвет подмножества пикселов, ранее окрашенных относительио ребра Р2-РЗ, инверсируется, т.е. меняется на цвет фона. На рие .1.б, это подмножество отмечено "I" (от англ. Inverse - перевернуть).

Закрашивание продолжается до окончания перебора всех ребер рис .1.г. При наличии буфера кадра порядок перебора ребер безразличен.
 
   

Экспериментальная часть

Для реализации алгоритма заполнения "по ребрам" на языке Си необходимо использовать следующие его функции:

  • drawpoly () - для вычерчивания полигона (замкнутой ломаной);
  • line () - для прорисовки горизонтальных линий;
  • putplxel () - для закраски пиксела;
  • getbkcolor () - для задания цвета фона;
  • bioskey () - для управления сменой ребра;
  • Initgraph (), closegraph () - для установки и закрытия графического режима.

Продемонстрировать работу алгоритма при наличии внутреннего полигона в большом полигоне, варьируя формы полигонов в соответствии с заданием и используя построчное заполнение областей.
